Configuración: Encontrando un compilador

Las primera cosa que necesitas hacer, antes de empezar en C, es estar seguro que tienes un compilador. ¿Que es un compilador, te preguntaras? Un compilador convierte el programa que escribes en un ejecutable que tu computador actualmente puede entender y correr. Si estas tomando un curso, probablemente te proporcionan uno a través de tu escuela. Si estas empezando por tu propia cuenta, la mejor apuesta es usar Code::Blocks con MinGW. Si estas en Linux, puedes usar GCC, y su estas en Mac OS X, puedes usar XCode. Si aún no lo ha echo, continúe y configure un compilador, lo necesitará para el resto del tutorial.

Introducción a C

Cada programa C completo empieza dentro de una función llamada main. Una función es una simple colección de comandos o declaraciones que hacen algo. La función main siempre es llamada cuando el programa se empieza a ejecutar. Desde main, nosotros podemos llamar otras funciones, ya sean escritas por nosotros mismos o por otros o usar funciones incorporadas en lenguaje. Para acceder a las funciones estándar que vienen con tu compilador, necesitas incluir una cabecera con la directiva #include. Lo que hace esto es tomar todo en el encabezado y copiarlo dentro de su programa. Observemos un programa trabajando:

1 /* comment */
2 #include <header.h>
3 int main(){
4
printf("Hello World!\n");
5
getChar();
6
return 0;
7 }

Vamos a mirar los elementos de el programa. El #include es una directiva del preprocesador que le dice al compilador que coloque el código desde el enzabezado llamado stdio.h dentro de tu programa antes de crear el ejecutable. Al incluir archivos de encabezado, puedes ganar acceso a muchas funciones diferentes: las funciones printf y getChar estan incluidas en stdio.h.

La siguiente línea importante es int main(). Esta línea le dice al compilador que hay una función llamada main, y que la función retorna un entero, por lo tanto int. Las llaves ({ y }) señalan el inicio y el fin de funciones y otros bloques de código. Si has programado en Pascal, conoceras esto como BEGIN y END. Incluso si no has programado en Pascal, esta es una buena manera de pensar acerca de su significado.

La función prinf el la manera estándar en C de mostar la salida en pantalla. Las comillas le dicen al compilador que quieres mostar un string o cadena literal tal como esta (casi). La secuencia '\n' es tratada en realizad como un solo caracter que representa una nueva línea (hablaremos acerca de esto mas tarde con mas detalle); para inciar, solo recuerca que hay unas pocas secuencias que, cuando aparecen en una string o cadena literal, en realizad no son mostradas literalmente por printf y que '\n' es una de estas. El efecto actual de '\n' es mover el cursor en la pantalla a la siguiente línea. Observe el punto y coma: le dice al compilador que estas al final de un comando, como una llamada a un función. Veras que el punto y coma ';' se usa para terminar muchas líneas en C.

El siguiente comando es getChar(). Este es otra llamada a una función: lee en un solo carácter y espera a que el usuario presione enter antes de leer el carácter. Incluyendo esa linea te da tiempo para ver el programa en ejecución.

Finalmente, al final del programa, devolvemos un valor desde main al sistema operativo usando la declaración return. Este valor de retorno es importante ya que puede usarse para decirle al systema operativo si nuestro programa tuvo éxito o no. Un valor de retorno 0 significa que la ejecución del programa tuvo exito.
